  Phase 2 — APIs

  Admin (/api/admin/ria/gouvernance/...)
  GET|POST /commissions            — liste + créer commission
  GET|PATCH /commissions/[id]      — détail + modifier membres
  GET|POST /commissions/[id]/reunions
  PATCH /commissions/[id]/reunions/[rId]  — signer présence, valider CR
  GET|POST /commissions/[id]/resolutions
  PATCH /resolutions/[id]          — vote + changement statut
  GET|POST /plans-actions
  PATCH /plans-actions/[id]        — mise à jour statut tâche
  GET|POST /dossiers               — workflow inter-commissions
  PATCH /dossiers/[id]             — APPROUVER | REJETER | DEMANDER_AJUSTEMENT |
  TRANSMETTRE
  GET /dashboard                   — KPIs agrégés DG
  GET /intelligence                — anomalies, alertes, portefeuilles en danger

  Membres (/api/membreCommission/...)
  GET /ma-commission               — commission scoped par rôle membership
  GET /ma-commission/reunions
  POST /ma-commission/reunions/[id]/signer
  GET|POST /ma-commission/observations

  Phase 4 — Pages Membres (portail utilisateur)

  app/dashboard/user/responsablesRIA/commissionsRIA/
  ├── page.tsx                        ← Dashboard membre (ma commission)
  ├── reunions/page.tsx               ← Mes réunions + signature
  ├── dossiers/page.tsx               ← Dossiers reçus / transmis
  ├── observations/page.tsx           ← Espace collaboration
  └── plans-actions/page.tsx          ← Mes tâches assignées

  Accès différencié selon le rôle :
  - PRESIDENT : valider rapports, créer résolutions, convoquer, affecter tâches
  - RAPPORTEUR_1 : consulter, préparer analyses, créer brouillons — pas de
  validation finale
  - RAPPORTEUR_2 : vérifier dossiers, modifier analyses, produire observations —
  pas de validation finale
