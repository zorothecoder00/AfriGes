Résumé du flux dans l'ordre logique

  1. Config → définir les taux (fait une seule fois / mis à jour si besoin)
  2. Investisseurs → créer le compte + profil RIA
  3. Fonds → enregistrer un dépôt → valider pour créditer le portefeuille
  4. Affectations → lier le portefeuille à un ou plusieurs clients avec un %
  5. Financements → créer un prêt client depuis le portefeuille affecté
  6. Recouvrement → suivre les remboursements, gérer les retards
  7. Scoring → réévaluer les classes de risque et scores de solvabilité
  8. Distributions → calculer et verser les bénéfices mensuels
  9. Commissions → rémunérer les agents de terrain sur leurs performances
  10. Reporting / Documents → générer les documents officiels et rapports
  11. BI / Alertes / Dashboard → surveiller la santé globale du réseau


 Module RIA — Guide complet du processus

  ---
  Vue d'ensemble du cycle de vie

  Configuration → Investisseurs → Portefeuilles → Fonds (dépôts)
         ↓
  Affectations (clients) → Financements → Recouvrement
         ↓
  Scoring → Bénéfices → Distributions → Commissions
         ↓
  Reporting / BI / Documents / Alertes

  ---
  1. Configuration (à faire en premier)

  Page : /ria/config

  C'est le point de départ obligatoire. Avant tout, l'admin définit les taux de
  répartition des bénéfices qui s'appliquent à tous les portefeuilles :

  ┌─────────────────────┬─────────────────────────────────────────┬─────────┐
  │        Champ        │                  Rôle                   │ Exemple │
  ├─────────────────────┼─────────────────────────────────────────┼─────────┤
  │ Taux généré         │ Rendement total produit par le capital  │ 10 %    │
  │                     │ engagé                                  │         │
  ├─────────────────────┼─────────────────────────────────────────┼─────────┤
  │ Taux distribué      │ Part versée à l'investisseur            │ 4 %     │
  ├─────────────────────┼─────────────────────────────────────────┼─────────┤
  │ Taux réinvesti      │ Part ajoutée au capital du portefeuille │ 4 %     │
  ├─────────────────────┼─────────────────────────────────────────┼─────────┤
  │ Taux fonds de       │ Réserve de garantie                     │ 2 %     │
  │ sécurité            │                                         │         │
  └─────────────────────┴─────────────────────────────────────────┴─────────┘

  La règle de validation : distribué + réinvesti + fonds sécurité = taux généré.
  Le bouton Enregistrer est grisé tant que l'équation n'est pas vérifiée. La
  page affiche aussi un historique de toutes les configurations passées — chaque
  save crée une nouvelle version, on peut voir l'évolution dans le temps.

  ---
  2. Création d'un Investisseur

  Page : /ria/investisseurs

  Le bouton "Nouvel investisseur" ouvre une modal avec deux modes :

  Mode "Nouveau compte"

  L'admin crée un compte utilisateur from scratch : nom, prénom, email,
  téléphone, adresse, profession, pays, notes. Un mot de passe temporaire est
  généré automatiquement. À la fin de la création, une popup affiche les
  identifiants (email + mot de passe temporaire) à transmettre à l'investisseur
  — il peut les copier avec le bouton copier. L'option "Créer un portefeuille
  principal" est cochée par défaut avec le nom "Portefeuille Principal"
  pré-rempli.

  Mode "Utilisateur existant"

  Si la personne a déjà un compte dans le système (un agent terrain, un RPV,
  etc.), on recherche par nom/email. On lui ajoute juste le rôle
  INVESTISSEUR_RIA et un profil RIA (profession, pays). Même option de créer un
  portefeuille en même temps.

  Liste des investisseurs

  La page affiche des cartes expandables : en cliquant sur une carte, les
  portefeuilles de l'investisseur se déroulent avec leurs KPIs (capital investi,
  disponible, bénéfices). Le bouton "Fiche" en bas à droite mène à la page
  détail.

  ---
  3. Fiche Investisseur

  Page : /ria/investisseurs/[id]

  Contient trois grandes sections :

  - Infos personnelles : nom, email, téléphone, profession, pays — avec bouton
  Modifier
  - Ses portefeuilles : liste expandable, chaque portefeuille montre les
  capitaux + bouton "Voir le portefeuille" qui mène à la page détail
  - Bouton "Ajouter un portefeuille" : ouvre une modal pour créer un nouveau
  portefeuille pour cet investisseur (référence auto générée PF-00001,
  PF-00002…, nom optionnel)

  ---
  4. Portefeuilles

  Page : /ria/portefeuilles (liste globale) et /ria/portefeuilles/[id] (détail)

  Page liste

  Tableau de tous les portefeuilles avec filtres par investisseur, capital,
  statut actif/inactif. Chaque ligne expandable montre les mouvements récents.

  Page détail d'un portefeuille

  C'est la fiche la plus complète du module — on y voit tout :
  - Répartition du capital : barre de progression visuelle (disponible / engagé
  / recouvré / bloqué)
  - Onglet Mouvements : journal chronologique de tous les flux (dépôts,
  retraits, financements, remboursements)
  - Onglet Affectations actives : les clients affectés avec leur classe de
  risque et encours
  - Onglet Financements : tous les prêts liés à ce portefeuille
  - Onglet Distributions : historique des bénéfices distribués

  ---
  5. Fonds — Dépôts et Retraits

  Page : /ria/fonds — 3 onglets : Dépôts | Retraits | Journal

  Dépôts (injection de capital)

  Le bouton "Nouveau dépôt" ouvre une modal : on choisit le portefeuille (liste
  déroulante avec nom de l'investisseur), on saisit le montant, le mode de
  paiement (virement, espèces, chèque…) et une note optionnelle.

  Le dépôt est créé au statut EN_ATTENTE. Dans la liste, chaque dépôt EN_ATTENTE
  a deux boutons :
  - ✓ Valider → statut VALIDE, le capital entre dans le portefeuille
  (capitalInvesti et capitalDisponible augmentent)
  - ✗ Rejeter → statut REJETE, aucun impact sur le capital

  Retraits (sortie de capital)

  Même logique : "Nouveau retrait" → on choisit portefeuille, montant, motif. Le
  retrait passe par EN_ATTENTE → Valider (= VALIDE, capitalDisponible réservé)
  → Payer (= PAYE, capitalDisponible effectivement déduit). Possibilité de
  Rejeter à tout moment avant PAYE.

  Journal

  Troisième onglet : historique filtrable de tous les MouvementFondsRIA — chaque
  opération validée génère automatiquement une ligne ici (dépôt, retrait,
  financement, remboursement, distribution). Filtres par type, sens
  (ENTREE/SORTIE), portefeuille.

  ---
  6. Affectations — Lier un investisseur à un client

  Page : /ria/affectations

  C'est l'étape qui dit : "ce portefeuille finance ce client à hauteur de X %".

  Le bouton "Nouvelle affectation" ouvre la modal avec :
  - Sélection du portefeuille : dès qu'on choisit, une barre de progression
  apparaît en dessous montrant les % déjà alloués aux autres clients de ce
  portefeuille + le total projeté
  - Sélection du client : champ de recherche live (nom, téléphone)
  - Allocation : toggle [Par pourcentage] / [Par montant] — on saisit une seule
  valeur, l'autre se calcule automatiquement et s'affiche en lecture seule
  - Classe de risque : A (très faible) → E (critique)

  Dans la liste

  Chaque ligne affiche la "Ligne de crédit" : une barre de progression encours /
  montantAlloue avec les couleurs verte (< 75 %), orange (75-99 %), rouge (100
  %+). On voit aussi le montant disponible restant.

  Le bouton Toggle (icône ToggleRight/ToggleLeft) en fin de ligne permet de
  désactiver / réactiver une affectation sans la supprimer — historique
  conservé.

  ---
  7. Financements — Les prêts aux clients

  Page : /ria/financements

  C'est le cœur opérationnel. Le bouton "Nouveau financement" ouvre la modal la
  plus complexe du module :

  1. Choix du portefeuille → charge les stats de capital disponible en temps
  réel
  2. Choix du client → si ce client a déjà une affectation active sur ce
  portefeuille, la ligne de crédit s'affiche automatiquement (montant alloué,
  encours actuel, disponible restant)
  3. Lien crédit client : champ optionnel pour rattacher ce financement à un
  crédit existant dans le système principal — utile pour la traçabilité croisée
  4. Montant : en FCFA, limité par le disponible de l'affectation
  5. Date d'échéance : date de remboursement prévue
  6. Taux / pourcentage RIA : la part RIA dans ce financement (si co-financement
  avec d'autres sources)
  7. Notes

  Le financement est créé au statut ACTIF. capitalDisponible du portefeuille est
  débité, capitalEngage augmente.

  Vue expandable dans la liste

  Chaque ligne de financement est expandable et montre :
  - La barre de remboursement (montantRembourse / montantFinance en %)
  - Les remboursements reçus avec dates
  - Le bouton "Ajouter un remboursement" directement inline
  - Les changements de statut : ACTIF → EN_RETARD → DEFAUT ou ACTIF → REMBOURSE

  Les statuts EN_RETARD et DEFAUT s'affichent en rouge, avec badge d'alerte.

  ---
  8. Recouvrement

  Page : /ria/recouvrement — 4 onglets : Vue globale | Clients | Alertes |
  Actions

  Onglet Vue globale

  4 KPIs : total encours, total remboursé, taux de recouvrement global, alertes
  actives. En dessous, un tableau par portefeuille avec le taux de recouvrement
  de chaque investisseur.

  Onglet Clients

  Tableau de tous les financements actifs/en retard avec les niveaux d'alerte :
  - J3 : 3-6 jours de retard — orange
  - J7 : 7-14 jours — orange foncé
  - J15 : 15-29 jours — rouge
  - J30+ : 30+ jours — rouge critique + badge "DÉFAUT IMMINENT"

  Chaque ligne est expandable : on voit le numéro de téléphone du client,
  l'agent terrain assigné, le lien vers le crédit associé.

  Onglet Alertes

  Vue concentrée uniquement sur les dossiers en retard, triés par gravité.
  Chaque alerte a un bouton "Action" qui ouvre un formulaire pour :
  - Enregistrer un contact client (note de relance)
  - Marquer un paiement partiel reçu
  - Planifier une visite terrain

  Onglet Actions

  Journal des actions de recouvrement enregistrées : relances, visites,
  promesses de paiement.

  9. Scoring

  Page : /ria/scoring — 2 sections : Scoring risque (A→E) | Scoring solvabilité
  (0→100)

  Scoring risque A→E (par affectation)

  La page liste toutes les affectations actives avec leur classe de risque
  actuelle (celle choisie manuellement) et la classe calculée automatiquement
  par l'algorithme.

  Le bouton "Recalculer" (icône Brain ✨)  relance l'algorithme sur toutes les  
  affectations. Le calcul se base sur :
  - Historique de paiement : nb d'échéances en retard sur le total
  - Retard actif : nb de jours de retard en cours
  - Taux de recouvrement : montant remboursé / montant financé
  - Fréquence : bonus si le client a eu ≥ 3 financements
  - Ancienneté : bonus si la relation dure > 6 mois

  Si la classe calculée diffère de la classe manuelle, une icône d'alerte jaune
  apparaît. L'admin peut cliquer "Appliquer" pour écraser la classe manuelle par
  la calculée.

  Scoring solvabilité 0→100 (par client)

  Section séparée avec un score composite sur 100 :
  - 40 pts : historique de paiement
  - 30 pts : taux de rotation (nb remboursements complets / total)
  - 30 pts : volume d'achat

  Le niveau textuel est dérivé du score : < 30 = CRITIQUE, 30-49 = MAUVAIS,
  50-64 = MOYEN, 65-79 = BON, ≥ 80 = EXCELLENT. Bouton "Mettre à jour tous" pour
  recalculer tous les scores d'un coup.

  10. Bénéfices

  Page : /ria/benefices

  Vue synthétique des bénéfices générés, distribués, réinvestis et mis en fonds
  de sécurité — par portefeuille et par période. Sert de vue de contrôle avant
  de déclencher les distributions.

  ---
  11. Distributions — Reverser les bénéfices

  Page : /ria/distributions

  Planifier une distribution

  Bouton "Planifier" → modal avec :
  - Choix du portefeuille
  - Mois / Année concernés
  - Dès que les deux sont choisis et que la configuration est présente, une
  prévisualisation s'affiche :
    - Capital de base engagé
    - Montant généré = capitalEngagé × tauxGénéré / 100
    - Montant distribué = montantGénéré × (tauxDistribué / tauxGénéré)
    - Montant réinvesti
    - Montant fonds de sécurité

  Statuts d'une distribution

  PLANIFIE → EN_ATTENTE_PAIEMENT → DISTRIBUE / REINVESTI

  Le bouton "Traiter" sur une distribution PLANIFIE ou EN_ATTENTE_PAIEMENT
  déclenche l'opération réelle : les montants sont crédités dans le portefeuille
  (capitalDisponible += distribué + réinvesti, capitalInvesti += réinvesti,
  fondSecurite += fonds).

  12. Commissions — Rémunération des agents

  Page : /ria/commissions

  Rémunère automatiquement 3 types de rôles selon leur performance de
  recouvrement :

  ┌───────────────┬───────────────────────────────────┐
  │     Rôle      │         Ce qui est mesuré         │
  ├───────────────┼───────────────────────────────────┤
  │ Agent Terrain │ Montant recouvré sur ses clients  │
  ├───────────────┼───────────────────────────────────┤
  │ Chef Agence   │ Recouvrement agrégé de son agence │
  ├───────────────┼───────────────────────────────────┤
  │ RPV Régional  │ Recouvrement régional             │
  └───────────────┴───────────────────────────────────┘

  Fonctionnement

  En haut de la page : sélecteur Mois / Année + bouton "Calculer les
  commissions". Le système parcourt tous les remboursements du mois, les groupe
  par responsable, applique le taux de base configuré et crée les lignes de
  commission au statut CALCULE.

  Cycle de validation

  CALCULE → APPROUVE → PAYE

  - Bouton "Approuver" : l'admin valide le montant calculé
  - Bouton "Payer" : marque comme effectivement versé (avec date de paiement)
  - Bouton "Annuler" : annule la commission (si erreur)

  Configuration des taux

  Un panneau en bas de page permet de définir le taux de base par rôle (ex :
  Agent Terrain = 2 %, Chef Agence = 1 %, RPV = 0.5 %) — ces taux sont
  activables/désactivables indépendamment.

  13. Reporting

  Page : /ria/rapports

  Génération de bulletins périodiques (mensuels / annuels) par portefeuille. Le
  bouton "Nouveau rapport" crée un rapport au statut BROUILLON. Chaque rapport
  est ensuite accessible via /ria/rapports/[id] où il peut être imprimé /
  exporté PDF avec window.print().

  15. Alertes Automatiques

  Page : /ria/alertes

  Surveillance en temps réel organisée en 3 sections :

  Clients en retard

  4 niveaux de gravité : J3, J7, J15, J30+. Chaque dossier liste le client, son
  téléphone, la référence du financement, le portefeuille concerné et
  l'investisseur.

  Investisseurs

  - Capital faible : portefeuilles dont capitalDisponible / capitalInvesti <
  seuil configuré (défaut 20 %)
  - Portefeuilles à risque : ceux ayant > 30 % d'affectations en classe D ou E
  - Rentabilité en baisse : comparaison taux de distribution mois en cours vs
  mois précédent, alerte si baisse > 10 %

  Direction

  - Impayés critiques : total montant en retard + défaut vs seuil (défaut 1 000
  000 FCFA)
  - Taux de défaut global : si > 10 % des financements actifs sont en problème

  16. BI — Business Intelligence

  Page : /ria/bi

  4 groupes d'analyses avancées en onglets :

  - Top Investisseurs : sous-onglets "Par capital investi" (bar chart) et "Par
  rentabilité/ROI" (% retour sur capital)
  - Top Clients : sous-onglets "Par volume financé" et "Par solvabilité" (score
  + niveau badge)
  - Top Agents : bar chart des agents terrain par montant recouvré
  - Top Régions : tableau des villes avec rendement colorisé vert/amber/rouge

  17. Indicateurs Stratégiques (dans le Dashboard principal)

  Page : /ria (dashboard)

  En plus des 6 blocs KPIs habituels, la section Indicateurs Stratégiques
  affiche :

  - Score de santé global : jauge circulaire 0→100 (vert ≥ 75, amber ≥ 50, rouge
  < 50)
  - NAV (valeur nette active = disponible + engagé + réinvesti + sécurité)
  - DSO (jours moyens de recouvrement)
  - Coût du risque et Taux de défaut avec alertes si seuils dépassés
  - 5 tableaux de rendement segmenté : par ville, par quartier, par secteur
  d'activité, par agence, par agent terrain
  - Graphique prévisions : 7 barres (J+30 à J+360) en mode Trésorerie ou
  Bénéfices (toggle)
